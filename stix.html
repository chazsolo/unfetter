Using STIX 2.0

The Unfetter|Discover project is built on the OASIS Open Standard called STIX. STIX is a standardized language for representing structure cyber threat information. Unfetter|Discover is prototyping the new STIX 2.0 specification that is still in Release Candidate.

STIX 2.0 was picked over the approved standard version 1.2.1 because STIX 2.0 uses JSON for its data representation rather than XML. This technology fits nicely with the NodeJS based software stack in Unfetter|Discover. Intending to just represent some of the data in STIX, the team eventually decided to implement all of Unfetter|Discover's backend and REST interfaces fully compliant with the STIX 2.0 schema.

Release 0.1 of Unfetter|Discover was a large monolithic program built on a Vagrant provisioned Virtual Machine. However, it became clear that the STIX 2.0 backend should be available to anyone wanting to store and retrieve STIX 2.0 data their own applications. Release 0.2 of Unfetter|Discover is being re-architected to support a micro-services model using Docker containers. All Unfetter Docker containers are available on the Unfetter organization on Docker Hub. The STIX 2.0 micro-services containers and software can be found on the GitHub repository cti-stix-services. The different containers in this repository are described below.

info

cti-stix-store

The Docker container cti-stix-store focuses on storage and retrieval of STIX 2.0 objects through a REST interface. The GET, POST, DELETE, and PATCH methods uses the json:api specification to address how to create, fetch, modify and delete STIX 2.0 objects. The data itself is stored based on the STIX 2.0 schemas. To run cti-stix-store, install Docker and run "docker up unfetter/cti-stix-store"

The REST interface that cti-stix-store uses can be explored through Swagger.

zoom_in

cti-stix-discover

Unfetter|Discover is focused on providing context to Cyber Threat Intelligence data by creating and managing relationships between that data. Thus, for Unfetter|Discover, the relationships between STIX 2.0 data is crucial to the product. Unfetter|Discover will soon release the cti-stix-discover Docker container from the cti-stix-services GitHub repo and \Unfetter on DockerHub.

This container uses a REST interfacace to query STIX 2.0 objects and their related objects. The cti-stix-discover container is a "stateless" microservice container and will rely on the cti-stix-store container to actually store the data.

dashboard

cti-stix-ui

Our third CTI STIX container will be cti-stix-ui. This container will provide a UI for creating, editing and deleting STIX 2.0 objects and relationships between them. This container will also be part of the cti-stix-services GitHub repo.
